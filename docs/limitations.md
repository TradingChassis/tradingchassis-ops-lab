# Limitations (v1)

This project is intentionally constrained for a finite v1.

- Single engine: NautilusTrader only
- Single example instrument: BTCUSDT only
- Intentionally simple toy strategy
- Local-first execution model
- No production-grade claim
- No low-latency claim
- No profitability claim
- No real capital required for v1
- Simplified reconciliation approach
- Simplified data handling assumptions

These limitations are deliberate to keep v1 focused on operational workflow quality rather than breadth or performance claims.
